Классы:

Entity		- объект, заданной формы, и имеющий позицию и направление.
StaticEntity	- объект, который не подчиняется физическим законам.
DynamicEntity	- объек, который подчиняется физическим законам.


Порядок обновления сущностей:

1.1)	ПУЛИ уничтожают всё, что могут. При уничтожении чего либо пуля уничтожается.
1.2)	КОРАБЛИ создают новые пули, уничтожают или возрождают пилотов.
1.3)	ТУРЕЛИ на карте создают новые пули.
1.4)	МИНЫ взрывают всё, что могут, в том числе себя. Мины при взрыве живут дольше одного тика.
1.5)	ЛАЗЕРЫ уничтожают всё, что могут. Лазеры после появления живут дольше одного тика.
1.6)	АКТИВНЫЕ ЛАЗЕРЫ карты уничтожают всё, чего косаются.
1.7)	НОЖИ уничтожают всё, чего косаются кроме корабля носителя.

2.1)	КОРАБЛИ, ПИЛОТЫ, АСТЕРОИДЫ общитывают столкновения между собой и картой,
	также учитывают воздействие на них генераторов гравитации.
2.2)	Прещёт скоростей и позиций всех сущностей.

Каждому пункту должна сответствует только одна функция.

В функции поведения объектов представляют собой цикл for с вложенными в него условиями
в которых МОЖЕТ изменятся состояние ТОЛЬКО основного объекта и вложенными циклами for
в нутри которых ДОЛЖНЫ изменяться состояния перебираемых объектов и МОЖЕТ изменяться состояние основного объекта.
Если состояния вложенных объектов не изменяются, такие объекты следует сделать основными и из их функций работать с основным объектом.

for()
{
	if()
	{
		изменение состояния основного объекта;
	}
	for()
	{
		изменение состояния вложенного объекта;
	}
}

Функции поведения не обрабатывают взаимодействия при столкновения.
Функции поведения могут добавлять и удалять вложенные объекты.
Функции поведения могут удалять основные объекты. В этом случае функция сразу переходит к следующему объекту.


Добавление и удаление объектов:

Для добавления у удаления объектов используются только функции из пространств имён Game::Add и Game::Delete.
При добавлении или удалении объекта изменяются только массивы объектов и счётчики объектов.
При этом, при вызове функции может изменяться любое количество массивов и соответствующих им счётчиков.

Размер массива, как и память для него выделяется при запуске игры и на протежении всей игры остаётся не изменной, поэтому количество объектов в нём не изменяется.
Удаление и добавление объектов происходит при помощи изменения флага exist конкретного объекта.
При добавлении объекта, перезаписываются все данные объекта.
В массыве объектов ищется первая пустая ячейка, после чего в неё записываются данные нового объекта.
При удалении объекта изменяется только его флаг.
